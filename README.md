## 1. Варіант індивідуального завдання
**Тема:** Розробка та автоматизоване розгортання вебапки для трекінгу задач.
**Опис завдання:** Необхідно розробити веб-застосунок з використанням бази даних PostgreSQL. Інфраструктурна частина вимагає написання bash-скрипта для повністю автоматизованого розгортання системи на чистій ОС.

## 2. Документація по розробленому веб-застосунку

### Призначення застосунку
Task Tracker — це мінімалістичний REST API сервіс для керування списком завдань. Дозволяє створювати нові задачі, переглядати їх список та змінювати статус на "виконано" (таке завдання по тз було). Також ендпоiнти для перевiрки стану вебапликейшена /health.


### Налаштування середовища для розробки 
Для локальної розробки потрібні встановлені Node.js та PostgreSQL.
1. Клонувати репозиторій: `git clone <посилання>`
2. Встановити залежності: `npm install`
3. Створити файл `.env` у корені проєкту та вказати підключення до бд:
   `DATABASE_URL="postgresql://user:password@localhost:5432/mywebapp?schema=public"`
4. Генерім клієнта та міграцію: 
   `npx prisma generate`
   `npx prisma db push`

### Як запустити застосунок 
Запуск у режимі розробки: `node server.js` (port = 8000).

## 3. Документація по API-ендпоінтах

| HTTP Метод | Ендпоінт (Шлях) | Призначення | Тіло запиту / Параметри |
| :--- | :--- | :--- | :--- |
| **GET** | `/tasks` | Вивести усі задачі (id, title, status, created_at) | - |
| **POST** | `/tasks` | Створити нову задачу | JSON: `{ "title": "назва" }` |
| **POST** | `/tasks/<id>/done` | Змінити статус задачі на "виконано" | `id` передається в URL |
| **GET** | `/health/alive` | Перевірка стану сервісу (завжди повертає OK) | - |
| **GET** | `/health/ready` | Перевірка підключення до бази даних | - |

## 4. Документація по розгортанню

### Образ вм та ресурси (мінімальні)
- **ОС:** Ubuntu 24.04 LTS.
- **Джерело:** Офіційний сайт [ubuntu.com](https://ubuntu.com/download).
- **Мінімальні ресурси ВМ:** 
  - CPU: 2 ядра
  - RAM: 2048 MB
  - Disk: 20 GB

### Вхід на ВМ та підготовка
При встановленні Ubuntu створюється користувач `app`. 
Усі подальші дії виконуються безпосередньо через консоль віртуальної машини:
1. Відкрити вікно віртуальної машини у VirtualBox.
2. Увійти в систему під створеним користувачем за замовчуванням.
3. Відкрити застосунок terminal всередині самої Ubuntu.
4. Нам потрібен інтернет тому пінгуем гугл для перевірки інтернета  `ping -c 3 google.com`, оскільки він потрібен для завантаження пакетів та скрипта.

### Завантаження та запуск автоматизації
Для повного розгортання системи достатньо виконати команди прямо в терміналі чистої вмкi:

```bash

wget [https://raw.githubusercontent.com/vitkovskiiy/task-tracker/main/setup.sh](https://raw.githubusercontent.com/vitkovskiiy/task-tracker/main/setup.sh) -O setup.sh

# 2. Даем чмод для исполнения 
chmod +x setup.sh

# 3. ЧЕРЕЗ СУДООО
sudo ./setup.sh
